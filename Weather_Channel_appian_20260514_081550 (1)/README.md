Generated by Legacy Modernizer for **Appian 25.4**.
Application alias: **WEC** — a new Appian application is created on import
(no existing app is overwritten).

## Import steps
1. **Appian Admin Console** → **Import**
2. Select this ZIP
3. *(Optional)* Also attach the `.properties` file to override environment-specific values
4. Click **Inspect**, review, then **Import**

> **Expected Inspect warning** — you will see *"Data store will be saved but
> will not be published: schema-validation missing table [...]"* (APNX-2-4053-006).
> This is normal for a fresh import — the tables don't exist yet. It's resolved
> automatically in step 1 of the checklist below (one click). **Do not** try
> to pre-create the tables with external DDL; Appian's Hibernate layer must
> own the schema per 25.4 guidance.

## Post-import checklist
The package intentionally **does not include record types** — Appian requires
a fully-wired source configuration for each record type, and generating that
correctly from legacy artifacts is unreliable enough to cause sync failures
(APNX-1-4521-016). Instead:

1. **Open the Data Store** (`WEC Data Store`) and click **Update Schema**
   to create the underlying tables.
2. **Create record types manually** — one per CDT — using Appian's recommended
   wizard (*Design → + New → Record Type → Data Store Entity*). Point each
   record type at the matching `WEC_*` entity in `WEC Data Store`.
3. **Link the site pages** (`WEC Weather Channel` site) to your new record
   types if you prefer record-backed navigation over the interface pages
   already included.
4. Install any plugins the SAIL interfaces reference (none are required for
   the default generated interfaces).
